Diskussion (Ergebnisse & Learnings)

In diesem Experiment wurde beobachtet, dass das LSTM-Modell kurze Wortfolgen aus den Trainingsdaten sehr präzise vorhersagen kann. Bei k=1 liegt die Treffersicherheit bei kleinen Datensätzen hoch, sinkt jedoch bei komplexeren Texten ohne erweitertes Training ab. Unter den Top k=5 oder k=10 Vorhersagen befindet sich das korrekte Wort signifikant häufiger. Ein niedriger Cross-Entropy Loss korreliert dabei spürbar mit einer flüssigeren autoregressiven Textgenerierung. Das Framework TensorFlow.js ermöglichte eine performante Ausführung dieser Vorhersagen in Echtzeit direkt im Browser.

Technische Dokumentation

Verwendete Frameworks:

  • TensorFlow (Python): Verwendet zum rechenintensiven Trainieren des Stacked LSTM-Netzwerks auf Cloud-GPUs in Google Colab.
  • TensorFlow.js (TFJS): Wird im Browser genutzt, um das konvertierte Modell clientseitig und ohne Server-Backend asynchron auszuführen.
  • Chart.js: Dient zur interaktiven, visuellen Darstellung der Top-Wortwahrscheinlichkeiten in einem Balkendiagramm.
  • Tailwind CSS: Ermöglicht ein schnelles, responsives Layout gemäß modernen UX/UI-Standards.

Technische Besonderheiten: Das Python-Modell musste via tensorflowjs_converter in ein Shard-basiertes Webformat übersetzt werden. Da der Web-Browser lokale JSON-Dateien aus Sicherheitsgründen sperrt, erfordert der Betrieb zwingend eine lokale Webserver-Umgebung (z.B. VS Code Live Server).

Fachliche Dokumentation

Implementierung der Logik: Der eingegebene Text (Prompt) wird beim Klick tokenisiert. Die letzten drei Wörter werden über das geladene word_to_idx-Dictionary in numerische Indizes übersetzt. Das TFJS-Modell liefert nach dem Forward Pass einen Tensor mit Softmax-Wahrscheinlichkeiten für jedes Wort im Vokabular. Über ein JavaScript-Sortierungsverfahren extrahieren wir die Top-Treffer.

Datenschutz-Diskussion (Rekonstruktion): Ja, die ursprünglichen Trainingsdaten lassen sich mittels des Modells teilweise rekonstruieren, wenn man bekannte Phrasen eingibt (Modell-Overfitting). Das stellt ein massives Datenschutzproblem dar, da neuronale Netze dazu neigen, sensible Informationen (wie Namen oder Passwörter) aus dem Trainingstext implizit zu speichern und über die autoregressive Generierung wieder preiszugeben.
